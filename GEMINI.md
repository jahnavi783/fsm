# FSM (Field Service Management) Mobile Application

## Project Overview

This is a Flutter-based mobile application for Field Service Management (FSM). It enables technicians to manage work orders, track parts, and communicate with the back office.

**Key Technologies:**

*   **Framework:** Flutter
*   **State Management:** `flutter_bloc`
*   **Dependency Injection:** `get_it` and `injectable`
*   **Routing:** `auto_route`
*   **Networking:** `dio` and `retrofit`
*   **Local Storage:** `hive_ce`
*   **Code Generation:** `build_runner`, `freezed`, `json_serializable`

## Getting Started

To run the application in development mode, use the following script:

```bash
./scripts/run_dev.sh
```

This will run the app with the `dev` flavor. Other available scripts for running the app are:

*   `./scripts/run_staging.sh`
*   `./scripts/run_prod.sh`

## Building the App

To build the Android APK for the `dev` environment, use the following script:

```bash
./scripts/build_android_dev.sh
```

Similar scripts are available for other environments and platforms:

*   `./scripts/build_android_staging.sh`
*   `./scripts/build_android_prod.sh`

## Project Structure

The project follows a feature-based structure. Each feature is located in its own directory under `lib/features`.

*   `lib/features/auth`: Authentication-related functionality.
*   `lib/features/work_orders`: Work order management.
*   `lib/features/parts`: Parts and inventory management.
*   `lib/features/calendar`: Scheduling and calendar view.
*   `lib/features/chat`: Communication and messaging.
*   `lib/core`: Core functionalities like networking, error handling, and models.
*   `lib/gen`: Generated code.

## State Management

The project uses the `flutter_bloc` package for state management. BLoCs are used to separate business logic from the UI. Each feature typically has its own BLoC.

## Routing

Navigation is handled by the `auto_route` package. Route definitions are located in `lib/core/router/app_router.dart`. The router is generated by `build_runner`.

## Code Generation

The project heavily relies on code generation for models, dependency injection, and routing. To run the code generator, use the following command:

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```
